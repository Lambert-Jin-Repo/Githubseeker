# Phase 11: Performance, UX & Features — Plan

**Date:** 2026-03-04
**Status:** Draft
**Depends on:** Phase 10 (complete)

---

## Overview

Phase 11 focuses on the highest-impact enhancements remaining after code quality hardening. Work is grouped into 3 milestones ordered by dependency and impact: Performance first (unblocks better UX), then UX improvements, then new features.

---

## Milestone 1: Performance — Faster AI Responses

**Goal:** Cut Phase 1 search time from ~30-50s to ~15-25s and deep dive from ~29s to ~15s.
**Effort:** 3-5 days
**Dependencies:** None (standalone)

| # | Task | Effort | Impact | Notes |
|---|---|---|---|---|
| 1.1 | Evaluate faster LLM alternatives | 1d | High | Benchmark Deepseek V3, Claude Haiku, Gemini Flash against MiniMax M2.5 on cost, latency, and tool-calling accuracy. Keep MiniMax as fallback. |
| 1.2 | Implement LLM provider abstraction | 1d | High | Create `lib/llm-provider.ts` interface so `lib/llm.ts` can swap providers without touching callers. Config via env var (`LLM_PROVIDER`). |
| 1.3 | Add streaming progress for tool calls | 0.5d | Medium | Send SSE events for individual tool call starts/completions during Phase 1, so the radar animation can show "Searching GitHub...", "Fetching metadata..." etc. |
| 1.4 | Cache GitHub metadata in Supabase | 0.5d | Medium | Store fetched repo metadata (stars, description, topics) with 24h TTL. Skip re-fetch for repos seen in recent searches. |
| 1.5 | Reduce deep dive LLM calls from 4 to 3 | 0.5d | Medium | Merge the two smallest prompt groups in V2 analyzer. Measure quality impact. |

### Success Criteria
- Phase 1 completes in <25s for specific queries, <35s for vague
- Deep dive analysis completes in <20s per repo
- No regression in output quality (spot-check 5 searches)

## Milestone 2: UX Improvements

**Goal:** Mobile-ready deep dive, keyboard power-user shortcuts, shareable search URLs.
**Effort:** 4-6 days
**Dependencies:** None (can run in parallel with Milestone 1)

| # | Task | Effort | Impact | Notes |
|---|---|---|---|---|
| 2.1 | Mobile-responsive deep dive page | 1.5d | High | Sidebar collapses to horizontal scrollable tabs on mobile. Section cards stack full-width. Test at 375px, 768px breakpoints. |
| 2.2 | Keyboard shortcuts | 1d | Medium | `j`/`k` to navigate repos in results table, `Enter` to expand, `/` to focus search, `Esc` to close. Add `?` shortcut overlay. Use a `useHotkeys` hook. |
| 2.3 | Shareable search URLs | 1.5d | High | Public `/s/[shortId]` routes that load saved results without auth. Generate short IDs (nanoid 8-char). Add "Copy Link" button to results page. Requires new `public_id` column on `searches` table. |
| 2.4 | Improve loading screen with tool-call progress | 0.5d | Medium | Wire Milestone 1 task 1.3 events into the orbital radar animation. Show strategy names as they execute. |
| 2.5 | Add toast notifications | 0.5d | Low | Replace `console.error` with user-visible toasts for feedback submit, export complete, link copied, errors. Use sonner or shadcn toast. |

### Success Criteria
- Deep dive page usable on iPhone SE (375px) with no horizontal scroll
- All keyboard shortcuts work, discoverable via `?` overlay
- Shared link loads results for non-authenticated visitors

## Milestone 3: Features

**Goal:** User accounts, search filtering, and comparison view.
**Effort:** 6-8 days
**Dependencies:** Milestone 2.3 (shareable URLs benefit from user accounts for ownership)

| # | Task | Effort | Impact | Notes |
|---|---|---|---|---|
| 3.1 | Google OAuth user accounts | 2d | High | Supabase Auth with Google provider. Migrate anonymous sessions: link `user_id` cookie to auth user on first login. Existing Google OAuth design doc available at `docs/plans/2026-02-26-google-oauth-design.md`. |
| 3.2 | Search history filter & search | 1d | Medium | Add text search input + mode filter (LEARN/BUILD/SCOUT) + date range picker to history page. Query via Supabase `.ilike()` + `.gte()`/`.lte()`. |
| 3.3 | Comparison view for repos | 2d | High | Select 2-4 repos from results table (checkbox), click "Compare". Side-by-side cards comparing: stars, activity, tech stack, code quality, community. Pulls from existing deep dive data. |
| 3.4 | PDF export | 1d | Medium | Add PDF option to export button. Use `@react-pdf/renderer` or `html2canvas` + `jsPDF`. Export deep dive report as formatted PDF with sections and charts. |
| 3.5 | Scheduled re-scans (stretch) | 2d | Low | Allow users to "watch" a search query. Cron job (Supabase Edge Function on `pg_cron`) re-runs search weekly, diffs results, sends email notification. Requires user accounts (3.1). |

### Success Criteria
- Google login works, previous anonymous searches visible after linking
- History page filters narrow results correctly
- Comparison view renders readable side-by-side for 2-4 repos
- PDF export produces a clean, printable document


## Risks & Mitigations

| Risk | Mitigation |
|---|---|
| LLM switch degrades output quality | Keep MiniMax as fallback behind feature flag; A/B test before full switch |
| Google OAuth complexity | Design doc already exists; use Supabase Auth (minimal custom code) |
| Mobile deep dive layout breaks with 12 sections | Progressive disclosure: collapse non-essential sections by default on mobile |
| PDF export is slow for large reports | Generate server-side in API route, return download link |
| Scheduled re-scans cost Serper credits | Limit to 3 watched queries per user; weekly cadence only |
